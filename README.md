# Maestro Kit — дирижёр с партитурой

Набор для Claude Code: **одна команда `/build` ведёт проект** от идеи до рабочего кода,
а знание проекта живёт в **вики на диске** — с индексом, журналом и уборкой. Дирижёр — процесс
(остановки ⛔, доказательства запуском, защита от стирания). Партитура — память (вики, из
которой любой новый чат входит в курс за минуту и не ищет одно и то же дважды).

Гибрид двух школ: vibe-build-kit (процессная дисциплина) + LLM-вики Карпати (дисциплина знания).

> 🔰 Совсем новичок? Открой **НАЧНИ-ЗДЕСЬ.md** — там всё по шагам, от установки Node.js.

## Ритуал (4 шага)

```bash
mkdir мой-проект && cd мой-проект
npx degit omikhaylov76-commits/maestro-kit/.claude .claude
# открой папку в Claude Code
/build
```

## Команды

| Команда | Зачем | Когда |
|---|---|---|
| `/build` | Дирижёр: правила → spec → архитектура → скиллы → аудит → цикл фич → аудит фазы | Основная работа |
| `/status` | «Где мы?» — доклад одной страницей, ничего не меняет | Начало любого чата |
| `/wiki` | «Запомни / найди» — карточка в базу знаний или ответ из неё | Появился факт/вопрос |
| `/handoff` | Передача смены: хэндофф + стартовое сообщение для нового чата | Конец сессии |

Больше команд нет сознательно: два пути к одному действию — начало винегрета.

## Философия (5 правил)

1. **Память на диске, не в чате.** Чат смертен, вики вечна: `wiki/` + `CLAUDE.md`.
2. **Остановки ⛔.** На каждой развилке дирижёр ждёт твоего «ок». Ты управляешь, не зная кода.
3. **Проверка с доказательством.** «Работает» = показан живой запуск, а не слова «всё ок ✅».
4. **Правило одного места.** Факт живёт ровно на одной странице, остальные ссылаются. Добыл
   внешний факт — сразу карточка. Никто не ищет одно и то же дважды.
5. **Шкаф не переполняется.** Индекс-однострочники, лимит размера страниц, Icebox для хотелок,
   чердак (attic) для устаревшего, вики-линт на границе каждой фазы.

## Что внутри

```
.claude/commands/   build · status · wiki · handoff
.claude/agents/     code-reviewer (ревью диффа без хардкода веток)
templates/          CLAUDE.md проекта · скелет wiki/ · .gitignore
library/            полки для пополнения: design-токены · skills · чек-листы аудитов
KIT.md              инструкция для человека (роль заказчика, ответы на ⛔, безопасность)
```

## Обновление набора в проекте

```bash
npx degit omikhaylov76-commits/maestro-kit/.claude .claude --force
```
`--force` перезапишет только `.claude/`. Твои `wiki/` и `CLAUDE.md` не трогаются.
