# НАЧНИ ЗДЕСЬ — пошагово для новичка

> Ничего страшного тут нет. Один раз пройдёшь — дальше всё по накатанной.
> Жирным — что нажать/набрать. Остальное — зачем это.

## Часть 1. Разовая подготовка компьютера (один раз в жизни)

**Шаг 1. Поставь Node.js** (это «моторчик», который запускает готовые проекты и команду установки).
Зайди на **nodejs.org**, скачай версию LTS, поставь как обычную программу.
Проверка: открой Терминал (на Mac: Launchpad → набери «Терминал»; на Windows: меню Пуск →
«PowerShell») и набери:

    node -v

Показал номер вроде `v24.1.0` — отлично. Пишет «command not found» — Node не встал, поставь заново.

**Шаг 2. Поставь Claude Code** — программу-кодера. Инструкция на сайте Anthropic:
docs.claude.com (раздел Claude Code). После установки в терминале должна работать команда `claude`.

## Часть 2. Новый проект (каждый раз ~2 минуты)

**Шаг 1. Создай папку проекта и зайди в неё.** В терминале набери (имя придумай своё,
латиницей, без пробелов):

    mkdir moy-proekt
    cd moy-proekt

`mkdir` — создать папку, `cd` — зайти в неё. Всё.

**Шаг 2. Установи набор Маэстро одной командой:**

    npx degit omikhaylov76-commits/maestro-kit/.claude .claude

Эта команда скачивает из интернета папку `.claude` с дирижёром и кладёт её в твой проект.
Точка в начале имени — обязательна, такие папки «скрытые», это нормально.

**Шаг 3. Открой папку в Claude Code.** Прямо из терминала набери:

    claude

(или открой программу Claude Code и выбери в ней эту папку).

**Шаг 4. Набери в чате:**

    /build

Всё. Дальше дирижёр сам: расспросит, что ты хочешь построить, предложит план и поведёт по шагам.

## Часть 3. Как отвечать дирижёру

Когда видишь значок **⛔** — дирижёр остановился и ждёт тебя. Твои ответы:

- **ок** — согласен, поехали дальше;
- **нет, …** — переделай вот так;
- **объясни простыми словами** — если непонятно (это нормально!);
- **реши сам** — доверяешь ему выбор.

Два правила безопасности: не набирай **/clear** сам (сотрёт память чата; если набрал — потом
снова `/build`), и если дирижёр собирается что-то **стереть навсегда** — он обязан попросить
тебя напечатать слово-подтверждение (например «удаляем»). Не просит — останови его.

## Часть 4. Каждый день

- Открыл чат → **/status** — «где мы» за минуту.
- Работаешь → **/build** — продолжит ровно с места остановки.
- Нашёл полезную статью или принял решение → **/wiki** — запомнит в базу знаний проекта.
- Заканчиваешь на сегодня → **/handoff** — соберёт передачку; завтра новый чат её подхватит.

Память проекта живёт в папке `wiki/` на диске — даже если чат пропал, ничего не потерялось:
новый чат → `/build` → продолжаем.

## Часть 5. Запустить то, что построили

Точные команды — в файле `CLAUDE.md` твоего проекта (дирижёр их туда записал). Для типичного
сайта: `npm install` (один раз), потом `npm run dev`, открой адрес из терминала
(обычно http://localhost:5173). Остановить — **Ctrl+C**.

## Частые проблемы

- **`/build` не находится** → папка `.claude` должна лежать в корне проекта и называться
  именно так, с точкой. Проверка: `ls -a` покажет её в списке.
- **`npx: command not found`** → не встал Node.js (Часть 1, Шаг 1).
- **Дирижёр запутался, повторяется** → открой новый чат и снова `/build`: память на диске.
- **Получилось «простенько»** → скажи дирижёру словами, как должно выглядеть — он внесёт в план.
