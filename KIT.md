# Maestro Kit — инструкция для человека

> Простым языком. Для тебя и любого, кому ты дашь этот набор.

## Что это
Одна команда `/build` в Claude Code ведёт проект от идеи до кода и не даёт ему развалиться.
Память проекта живёт на диске в папке `wiki/` — потерять чат не страшно.

## Твоя роль
Ты — заказчик. Рассказываешь, чего хочешь, и отвечаешь на остановках ⛔. Понимать код,
архитектуру и git не нужно. Непонятно — пиши «объясни простыми словами» или «реши сам».

## Значок ⛔ — главное
Дирижёр остановился и ждёт тебя. Ответы: **«ок»** — дальше; **«нет, …»** — переделает;
**«ок, но …»** — поправит и снова спросит; вопрос — объяснит и снова спросит;
**«реши сам»** — решит и пойдёт дальше (кроме стирающих действий).
Проскочил без остановки — напиши: «стоп, ты не остановился».

## Два правила безопасности
1. Не набирай `/clear`, не спросив дирижёра. После `/clear` — обязательно снова `/build`.
2. Всё, что стирает работу безвозвратно, требует напечатать слово-подтверждение
   (например «удаляем»). Не просит слово, а собирается стирать — останови его.
Не жми «ок» вслепую на стирающих действиях.

## Ежедневные команды
- Открыл чат → `/status` — доклад «где мы» за минуту.
- Работать → `/build` — сам продолжит с места остановки.
- Нашёл статью/сервис/принял решение → `/wiki` — запомнит навсегда, дважды искать не придётся.
- Заканчиваешь → `/handoff` — соберёт передачку для следующего чата.

## Запуск результата
Точные команды запуска твоего проекта записаны в `CLAUDE.md` (дирижёр спросит их на старте).
Типично для веба: `npm install` → `npm run dev` → адрес из терминала → стоп Ctrl+C.

## Если что-то не так
- `/build` не находится → папка `.claude` (с точкой!) должна лежать в корне проекта.
- Дирижёр «поплыл» → новый чат → `/build`: память на диске, ничего не потеряно.
- Вышло «простенько» → внешний вид не проговорили в spec: скажи словами, как должно
  выглядеть, или попроси фичу «улучшить внешний вид».
- Ответы зависают → попроси работать короткими порциями (это и так правило набора).
