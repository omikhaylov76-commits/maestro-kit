## ПРОЦЕСС (не вырезать — на этом держится дисциплина)
- Работа ведётся через /build (дирижёр), статус — /status, знания — /wiki, смена — /handoff.
- ⛔ — остановка: дирижёр ждёт «ок»/«нет»/«объясни»/«реши сам». «Реши сам» не действует
  на стирающие действия — им нужно слово-подтверждение (например «удаляем»).
- «Готово» только с доказательством: дифф по плану, вывод сборки, ЖИВОЙ запуск.
- Один под-шаг за раз; максимум 3 попытки чинить один затык, дальше — предложение отката.
- Код — на ветке `task/<id>` (зелёные тесты + code-reviewer + ⛔ → слияние в main со словом);
  доки/chore — прямо в main. Коммит — только после «ок».
- Короткие порции: большие ответы делить, файлы читать по одному.

## ВИКИ (память проекта)
- Вход в проект: index → хвост log → roadmap → активный progress. Всё дерево не сканировать.
- Правило одного места: факт — на одной странице, остальные ссылаются.
- Добыл внешний факт — сразу карточка (entities/summaries) + index + log.
- Страница ≤150 строк; однострочник в index ≤100 символов; хотелки — в Icebox;
  устаревшее — в attic. Frontmatter: type/title/tags/updated/sources.
